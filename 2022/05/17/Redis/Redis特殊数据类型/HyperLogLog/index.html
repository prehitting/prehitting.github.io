HyperLogLog
2022-05-17|RedisRedis特殊数据类型

什么是基数

一个集合中不重复的元素个数，可以接收误差

简介

  • Redis 在 2.8.9 版本添加了HyperLogLog结构
  • 基数统计算法
  • 优点：占用内存固定，2^64不同的元素只需要12KB内存！如果要从内存角度来比较，Hyperloglog首选
  • 存在0.81%的错误率，如果不允许容错，使用set即可
  • 网站的UV（一个人访问一个网站多次，但算作一个人）
    • 传统统计方式：set存储用户id，然后就可以统计set中的元素作为标准判断
    • 用传统方式保存大量用户id会比较麻烦

命令

1[PFADD key element [element …]]添加指定元素到 HyperLogLog 中。
2[PFCOUNT key [key …]] 返回给定 HyperLogLog 的基数估算值。
3[PFMERGE destkey sourcekey [sourcekey …]]将多个 HyperLogLog 合并为一个 HyperLogLog
